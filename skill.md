# Skill: CRM 综合运营数据助手 (Performance & Call Analysis)

## 1. 核心能力定义
通过执行 `main/crm_tool.py` 脚本，Agent 可实时获取并关联 CRM 系统的两类核心数据，并支持**自定义日期范围**的行为分析：
1. **业绩表现 (`perf`)**: 员工的实时排名、销售额及小组归属。
2. **行为效率 (`call`)**: 员工在**特定时间段内**的拨号频次、累计时长及开工时间（First Call）。

## 2. 技能指令集 (CLI Reference)
Agent 应根据用户提到的时间关键词（如“昨天”、“4月20日”）自动计算日期参数。

| 查询场景 | 执行指令格式 | 参数说明 |
| :--- | :--- | :--- |
| **实时业绩查询** | `python main/crm_tool.py perf [姓名]` | 姓名可选，默认 Top 5 |
| **今日通话统计** | `python main/crm_tool.py call [姓名]` | 默认查询当日数据 |
| **特定日期通话** | `python main/crm_tool.py call [姓名] [日期] [日期]` | 日期格式：YYYY-MM-DD |
| **日期区间分析** | `python main/crm_tool.py call [姓名] [开始日期] [结束日期]` | 用于周报/月报分析 |

## 3. Agent 运行规范 (Prompting Guide)

### **A. 智能日期处理**
当用户描述模糊时间时，Agent 需将其转化为脚本可识别的格式：
- **“昨天”**: 获取当前日期并减 1 天。
- **“本周”**: 计算本周一至今的日期区间。
- **“4月1号到10号”**: 转化为 `2026-04-01 2026-04-10`。

### **B. 深度分析逻辑 (Insights)**
- **横向对比**: 关联同一员工在不同日期的通话时长，判断其状态是否稳定。
- **效率评估**:
  - *高时长低业绩*: “该员工在 [日期区间] 通话极长但转化率低，可能需要话术辅导。”
  - *晚开工提醒*: “该员工在过去 3 天的 First Call 均晚于 10:00，建议关注考勤。”

## 4. 交互示例 (Conversational Scenarios)

**场景 A：查询特定日期的表现**
* **User**: "帮我看看 Panawat 昨天的通话时长。"
* **Agent**: (执行 `python main/crm_tool.py call Panawat 2026-04-23 2026-04-23`) "Panawat 昨天（4月23日）的表现如下：总通话 **145分钟**，有效率 **32%**，第一通电话于 **09:05** 拨出。"

**场景 B：跨日期区间分析**
* **User**: "对比一下王小明这周前三天的通话量。"
* **Agent**: (执行 `python main/crm_tool.py call 王小明 2026-04-20 2026-04-22`) "王小明在本周前三天（20-22日）累计通话 **420分钟**，日均拨打 **45次**。数据显示其工作量非常稳定。"

**场景 C：多维综合评估**
* **User**: "Panawat 最近表现怎么样？"
* **Agent**: (先执行 `perf` 查排名，再执行 `call` 查近 3 日数据) "Panawat 目前业绩稳居全组 **第2名**。从通话数据看，他近 3 天平均每天通话超过 **160分钟**，且首通电话均在 **09:10** 前拨出，属于高产且自律的员工。"

## 5. 维护说明 (Maintenance)
- **参数占位**: 若仅查询日期而不指定姓名，请在 `[姓名]` 位置传入 `None`。
- **数据映射**: 脚本已适配 `start_date` 与 `end_date` 字段，确保 POST 请求参数名与 CRM 网页源码一致。
